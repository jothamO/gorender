# Security Policy (Draft)

## Supported Scope

Security reports are accepted for:

- `gorender` CLI/runtime code
- `gorendersd` server code
- release artifacts and supply chain concerns

## Reporting

- Do not open public issues for sensitive vulnerabilities.
- Use private disclosure channel (to be configured before public launch).

Report should include:

- affected version/commit
- reproduction steps
- impact assessment
- suggested mitigation if known

## Response Targets (Draft)

- Initial triage: within 3 business days
- Status update: within 7 business days
- Patch/mitigation timeline shared after validation
